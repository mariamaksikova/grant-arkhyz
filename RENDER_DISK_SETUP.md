# Настройка персистентного диска на Render.com

## Где найти диск в Render Dashboard:

1. **Войдите в Render Dashboard**: https://dashboard.render.com
2. **Выберите ваш сервис**: `grant-arkhyz`
3. **Перейдите в раздел "Disks"** (в боковом меню слева)
4. **Проверьте наличие диска**:
   - Name: `database`
   - Mount Path: `/opt/render/project/src/data`
   - Size: `1 GB`

## Если диска нет:

### Вариант 1: Через Dashboard (рекомендуется)
1. В настройках вашего сервиса найдите раздел **"Disks"**
2. Нажмите **"Add Disk"** или **"Create Disk"**
3. Заполните:
   - **Name**: `database`
   - **Mount Path**: `/opt/render/project/src/data`
   - **Size**: `1 GB`
4. Сохраните изменения

### Вариант 2: Через render.yaml
Файл `render.yaml` уже настроен, но Render может не применять его автоматически.
Убедитесь, что:
- Файл находится в корне репозитория
- Render использует этот файл для деплоя (проверьте в настройках сервиса)

## Проверка работы диска:

После настройки диска проверьте логи приложения. Должно быть:
```
✓ Подключено к базе данных SQLite
✓ Путь к базе данных: /opt/render/project/src/data/database.db
✓ Диск существует: true
```

## Важно:

- Диск должен быть создан **ДО** первого деплоя с базой данных
- Если диск создан после, старые данные будут потеряны
- После создания диска нужно перезапустить сервис

## Альтернативное решение:

Если персистентный диск не работает, можно использовать:
- **PostgreSQL** (управляемая база данных на Render)
- **Внешний сервис базы данных** (например, Supabase, Railway)
